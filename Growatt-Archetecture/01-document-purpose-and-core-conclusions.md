# 执行摘要与核心结论

## 执行摘要

本文档从平台架构视角，对当前企业平台进行统一整理与可视化表达，适用于以下外部沟通场景：

- 平台架构介绍
- 第三方技术交流
- 第三方接入沟通
- 生态合作伙伴能力说明
- 协议与设备建模口径对齐

本文重点覆盖以下内容：

- 平台总体分层架构
- 北向接口与云边通信关系
- 南向协议抽象与设备接入关系
- 电池与逆变器的从属建模关系
- 面向不同角色的分图视角

------

## 核心结论

### 平台组成

当前企业平台主要包括以下核心组成部分：

#### 云端（Cloud / Server）

负责：

- 设备管理
- 遥测数据处理
- 告警处理
- 控制服务
- 对外 API 服务

#### 对外 API

面向第三方平台提供两套接口：

- **Legacy API**
  - Token 鉴权
- **OpenAPI**
  - OAuth 2.0 鉴权

#### 边缘接入（Collector / Gateway）

负责：

- 设备数据采集
- 云边通信
- 协议转发
- 北向数据上传
- 南向设备接入

#### 协议抽象层（Protocol Abstract Layer / ARM Platform）

位于 Edge Connectivity Layer 与 Device Layer 之间，负责：

- 南向协议适配
- 设备差异收敛
- 状态模型统一
- 标准协议与非标协议并存管理

#### 设备层（Device Layer）

当前主要设备系列包括：

- **XH 系列**：Hybrid Inverter
- **SPH 系列**：Hybrid Inverter
- **X 系列**：PV Inverter
- **SPF 系列**：Off-line Inverter

------

### 北向通信链路

平台在云端 Server 与采集器之间当前存在两条通信路径：

#### 1）`0x04` 分类分级链路

- 用于快速刷新
- 刷新周期约 **1 min**
- 面向较高实时性的数据上送场景

#### 2）`0x03` 普通查询链路

- 用于 legacy 查询
- 刷新周期约 **5 min**
- 面向兼容性和常规查询场景

------

### 南向协议现状

采集器南向通过协议抽象层统一接入设备，当前存在两类协议路径：

#### 1）VPP Protocol 2.04

- 标准化版本
- 已完成标准化接入的设备包括：
  - **XH 系列**
  - **SPH 系列**
  - **X 系列**

#### 2）RTU Protocol

- 非标/legacy 版本
- 当前主要适用于：
  - **SPF 系列**

------

### 电池建模原则

电池**不作为平台独立接入对象**，而是通过逆变器进行管理、状态上送与控制执行。

对应关系如下：

- **XH 系列** → **HV Battery（高压电池）**
- **SPH 系列** → **LV Battery（低压电池）**
- **SPF 系列** → **LV Battery（低压电池）**
- **X 系列** → 纯光伏逆变器，不直接管理电池

这一定义适用于：

- 架构设计
- 设备建模
- 协议抽象
- API 语义设计
- 遥测字段归属定义
